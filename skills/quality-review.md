# Quality Review

Use this skill when reviewing generated website output.

## Checkpoints

- Does the site satisfy the original brief?
- Is there a clear user journey from first impression to next action?
- Are required Vue/Vite files present?
- Are dependencies declared?
- Is the interface responsive without text overlap?
- Are there obvious accessibility issues such as missing labels, low contrast, or unclear link text?
- Is the implementation specific rather than generic?

## Review Output

- Approve only when the site is coherent and runnable.
- Request changes when a fix is specific, useful, and feasible in one revision pass.
- Keep feedback short and actionable.
